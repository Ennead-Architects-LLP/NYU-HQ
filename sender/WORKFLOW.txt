KEY DECISIONS
═══════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────┐
│  Why rename HTML to index.html?                                         │
│  • GitHub Pages serves index.html at root URL                           │
│  • Users access site without filename in URL                            │
│  • Report generator can use any HTML filename                           │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│  Why keep supporting files' original names?                             │
│  • HTML references them by name: href="styles.css"                      │
│  • No need to rename - they're not accessed directly                    │
│  • Predictable naming for report generators                             │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│  Why upload ALL supporting files?                                       │
│  • Report might reference any combination of CSS/JS/JSON                │
│  • Simpler logic - no need to parse HTML to find references             │
│  • Report generator controls what's in the folder                       │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│  Why choose latest HTML by modification time?                           │
│  • Report generators create timestamped files                           │
│  • Most recent = most accurate data                                     │
│  • Backward compatible with single HTML setup                           │
└─────────────────────────────────────────────────────────────────────────┘
